# 대전 교통사고 위험지역 분석 학습 앱 레포 설명

이 레포는 고등학교 데이터과학 수업에서 대전 교통사고 공공데이터를 지도, 차트, 상관관계, 예측 시나리오로 탐구할 수 있게 만든 정적 웹앱입니다. 설치형 서버나 데이터베이스 없이 `index.html`, `app.js`, `styles.css`, CSV 파일만으로 동작하도록 구성했습니다.

## 핵심 목표

학생이 단순히 "어디가 위험한가"를 보는 데서 멈추지 않고, 왜 그 지점이 위험하게 계산되었는지 데이터 항목과 모델 가정으로 설명하게 만드는 것이 목표입니다.

- 사고 건수, 사상자 수, 중상자 수처럼 원자료에 가까운 항목을 확인합니다.
- 어린이, 고령자, 야간, 출퇴근, 속도 제한, 교통량 같은 해석 변수를 비교합니다.
- 같은 모델을 여러 데이터셋에 적용했을 때 결과와 해석이 달라지는 이유를 토론합니다.
- 시나리오를 바꾸며 정책 개입이 점수에 어떤 영향을 주는지 살펴봅니다.

## 앱 화면 구성

`index.html`은 화면의 뼈대입니다.

- 상단 조건 영역: 데이터셋, 구/군, 시간 관점, 속도 제한, 예측 시나리오를 선택합니다.
- 데이터셋 안내 영역: 현재 선택한 데이터가 어떤 목적으로 만들어졌는지와 해석 주의점을 보여줍니다.
- KPI 영역: 평균 위험도, 총 사고 건수, 중상 이상, 고위험 지점 수를 요약합니다.
- 지도 영역: Leaflet과 OpenStreetMap으로 사고다발지역 마커와 밀도 히트맵을 표시합니다.
- 차트 영역: 지역별 집계, 시간대 패턴, 속도와 사고 규모, 상관관계를 보여줍니다.
- 예측 결과 해설: 위험도 상위 지점과 항목별 해석 문장을 보여줍니다.

## 데이터셋

현재 앱에서 선택할 수 있는 데이터셋은 다음과 같습니다.

- `수업용 샘플 데이터`: 수업 설명을 위해 대전 5개 구의 위험 요인이 고르게 보이도록 만든 예제입니다.
- `KOROAD 2024 사고다발지역`: 한국도로교통공단 지자체별 사고다발지역 원자료를 앱 형식에 맞춘 자료입니다.
- `KOROAD 보행자 사고다발지역`: 보행자 피해 집중 지점을 분석하기 위한 자료입니다.
- `KOROAD 노인 보행자 사고다발지역`: 고령 보행자 안전을 중심으로 해석하는 자료입니다.
- `KOROAD 어린이보호구역 사고다발지역`: 어린이보호구역과 등하교 시간대 관리를 중심으로 해석하는 자료입니다.

데이터셋이 다르면 같은 위험도 모델을 쓰더라도 결과가 달라집니다. 예를 들어 전체 사고다발지역은 사고 규모와 교통량이 큰 교차로가 두드러지고, 노인 보행자 자료는 시장과 역 주변 고령 보행자 안전 문제가 더 크게 드러납니다.

## CSV 컬럼

직접 만든 CSV를 넣으려면 아래 필수 컬럼을 맞춰야 합니다.

```text
district,area,lat,lng,accidents,casualties,serious,children,elderly,night,commute,pm,weatherRisk,speedLimit,schoolZone,roadType,signalDensity,trafficVolume
```

선택 컬럼은 다음과 같습니다.

```text
id,note
```

`night`, `commute`, `weatherRisk`, `signalDensity`, `trafficVolume`은 `0.35`처럼 0~1 비율로 넣거나 `35`처럼 퍼센트 값으로 넣을 수 있습니다. 앱은 1보다 큰 값을 자동으로 100으로 나누어 비율로 처리합니다.

## 전처리

`tools/preprocess_koroad_datasets.py`는 `data/raw/`의 한국도로교통공단 JSON 원자료를 읽어 앱에서 바로 쓸 수 있는 CSV로 변환합니다.

```bash
python3 tools/preprocess_koroad_datasets.py
```

전처리 과정에서는 원자료의 위치, 사고 건수, 사상자 수, 중상 이상 인원, 좌표를 우선 사용합니다. 원자료에 없는 수업용 설명 변수는 구별 통계, 위치 키워드, 사고다발지역 유형을 바탕으로 파생합니다. 따라서 이 앱의 결과는 정책 결정을 위한 확정 분석이 아니라 데이터과학 수업용 모델링 사례로 읽어야 합니다.

## 위험도 모델

위험도 점수는 `app.js`의 `riskScore()`에서 계산합니다.

모델은 다음 항목을 0~1 범위로 정규화하거나 비율로 읽은 뒤 가중 합산합니다.

- 사고 건수
- 중상 이상 인원
- 사상자 수
- 어린이와 고령자 관련 값
- 야간 비중
- 출퇴근/등하교 비중
- 제한속도
- 교통량 지표
- 기상 취약도

이 점수는 정답이 아니라 비교 도구입니다. 수업에서는 "점수가 높다"보다 "어떤 변수가 점수를 높였는가", "이 가중치가 타당한가", "다른 데이터셋을 쓰면 해석이 어떻게 달라지는가"를 더 중요하게 다루어야 합니다.

## 지도 데이터 확인

지도는 실제 행정구역 경계 파일을 사용하지 않고 OpenStreetMap 배경지도 위에 원자료 좌표를 표시합니다. 예전에 임의로 만든 구 경계선이 실제 경계처럼 보일 수 있어 현재는 기본적으로 표시하지 않습니다.

앱은 좌표가 대전 범위 안에 있는 행만 지도에 표시합니다.

- 위도: `36.25` 이상 `36.5` 이하
- 경도: `127.25` 이상 `127.5` 이하

CSV를 추가했는데 지점이 보이지 않으면 좌표가 뒤바뀌었는지, 위도/경도 값이 대전 범위를 벗어났는지 먼저 확인합니다.

## 새 데이터를 넣었을 때 해석 방식

새 CSV를 업로드하면 앱은 같은 모델로 위험도를 다시 계산합니다. 이때 해석은 다음 순서로 진행합니다.

1. 출처와 집계 기준을 확인합니다.
2. 좌표가 지도에 올바르게 찍히는지 확인합니다.
3. 사고 건수, 중상, 어린이, 고령자, 야간, 출퇴근, 속도 제한 중 어떤 항목이 높은지 봅니다.
4. 상위 위험 지점의 `note`와 항목별 해석 문장을 비교합니다.
5. 시나리오를 바꾸어 점수가 내려가거나 유지되는 이유를 설명합니다.

GPT 같은 생성형 AI에 해석을 맡길 때도 이 순서가 중요합니다. 데이터셋 이름, 컬럼 정의, 상위 지점, 위험도 산식, 한계를 함께 제공해야 결과를 과장하지 않고 수업용 분석으로 해석할 수 있습니다.
